Release Notes for WESNet Designs By Invitation v 01.00.01

Minimum DotNetNuke Framework Version: 07.03.03

Issues Fixed in This Release:

  • Clicking any action (Approve|Disapprove|EditApprove) in moderation requested notification results in error and action is not performed.
  • Null Reference Exception occurs when opening the Manage Invitations control if any invitation's inviting user account has been hard deleted.
  • Adjusted several rules in module.css to fix positioning issues in module's custom settings.
  • Fixed extraneous characters in processor form "Wecoome . . . " message displayed when accepting an invitation. Corrected ProcessInvitation.ascx.resx resource file.
  • Fixed failure of confirm dialog box opening when deleting expired invitations displayed in the Manage Invitations control.

Enhancements and Other Changes:

  • Added "Load Default" link button above Invitation Email Body text/html editor in module's custom settings to restore (from SharedResources.resx) the default invitation body html.
  • When site user settings have enabled "Use Email as Username", the accept invitation processor form now correctly displays 'Email Address' as the caption for the Username field.
  • When site user settings have enabled "Use Email as Username" and modification of Username is allowed upon acceptance, the Username/Email field's validation regular expression will be the site's email rather than username regex.

Date of Release: February 27, 2015
